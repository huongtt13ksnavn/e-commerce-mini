## Affected Pages/Routes
- `POST /api/auth/register` ‚Äî user registration, happy path + duplicate email
- `POST /api/auth/login` ‚Äî JWT generation, wrong credentials, rate limiting (429 on 6th req/min)
- `GET /api/auth/me` ‚Äî JWT claims read, 401 without token
- `GET /api/products` ‚Äî paginated list with search, empty result
- `GET /api/products/{id}` ‚Äî single product, 404 for unknown
- `POST /api/products` ‚Äî admin only, 403 for user JWT, 401 for no auth
- `PUT /api/products/{id}` ‚Äî admin price/name update, 404 for unknown
- `PATCH /api/products/{id}/deactivate` ‚Äî admin only, 204
- `GET /api/cart` ‚Äî current user's cart (empty for new user), items present after add
- `POST /api/cart/items` ‚Äî add item, inactive product ‚Üí 400
- `DELETE /api/cart/items/{productId}` ‚Äî remove item, 204
- `DELETE /api/cart` ‚Äî clear cart, 204
- `POST /api/orders` ‚Äî place order from cart (happy path), empty cart ‚Üí 400, inactive product ‚Üí 400
- `GET /api/orders` ‚Äî list for current user
- `GET /api/orders/{id}` ‚Äî detail, 404 for unknown, 404 (not 403) for other user's order
- `PATCH /api/orders/{id}/cancel` ‚Äî cancel order, 400 if already completed
- `GET /healthz` ‚Äî 200 healthy, 503 if DB down

## Key Interactions to Verify
- Full browse-to-order flow: login ‚Üí browse products ‚Üí add to cart ‚Üí place order ‚Üí verify cart cleared
- `OrderPlaced` domain event fires on order placement (verify via notification handler log or test spy)
- Rate limiter blocks 6th login attempt within 1 minute from same IP ‚Üí 429
- Admin JWT required for all product write endpoints; user JWT ‚Üí 403
- `IDistributedCache` (Redis) serves product reads; cache invalidated after product update

## Edge Cases
- `POST /api/orders` with empty cart ‚Üí 400 ProblemDetails `{ title: "Cart is empty" }`
- `POST /api/orders` with inactive product in cart ‚Üí 400 `{ title: "Product {id} is no longer available" }`
- `GET /api/orders/{id}` with different user's JWT ‚Üí 404 (never 403)
- FluentValidation failure (e.g., invalid email on register) ‚Üí 422 ProblemDetails with field errors
- `DomainException` subclass ‚Üí 400 ProblemDetails
- Unexpected exception in handler ‚Üí 500 ProblemDetails (no stack trace in response)
- New user GET /api/cart ‚Üí empty cart (not null/404)
- Deactivated product ‚Üí cannot be added to cart (400)

## Critical Paths
1. Register ‚Üí Login ‚Üí Browse products (paginated) ‚Üí Add item to cart ‚Üí Place order ‚Üí Cart cleared ‚Üí OrderPlaced fired
2. Admin login ‚Üí Create product ‚Üí Deactivate product ‚Üí Verify product not addable to cart
3. Register two users ‚Üí User A places order ‚Üí User B tries GET /api/orders/{userA_orderId} ‚Üí 404

## Test Infrastructure
- xUnit + WebApplicationFactory<Program> + Testcontainers (PostgreSQL per test collection)
- Naming: `MethodName_Scenario_ExpectedResult` (e.g., `PlaceOrder_WithEmptyCart_Returns400`)
- Test helpers: `TestAuthHelper` to get JWT tokens for seed users (admin + regular user)
- No mocking ‚Äî all tests hit real DB via Testcontainers
- CI: GitHub Actions ubuntu-latest, first run ~60-90s for Testcontainers image pull
