## Vision

### 10x Check
The 10x version of this project isn't more features — it's depth plus distribution. A deployed live URL, a technical blog post documenting the AI-assisted 5-day build workflow (with commit timestamps showing what was built when), and OpenTelemetry traces making the app actually observable. The code is the proof; the post is the multiplier. Every senior .NET engineer who reads that post either interviews you or shares it.

### Platonic Ideal
The user opens a GitHub link, reads the README in 2 minutes, runs `docker compose up`, hits the Scalar UI, walks through the full browse-to-order flow, opens two files (a command handler and a domain aggregate), sees clean DDD and CQRS in action with proper value objects and domain events, reads one ADR that answers a question they were about to ask, and thinks: "This person has shipped production systems." They forward the link to their hiring manager.

## Accepted Scope (added to this plan)

- `GET /healthz` — health check endpoint using `AddHealthChecks().AddNpgSql()`; returns 200 OK or 503; Docker Compose `depends_on: api: condition: service_healthy` added for the API container; Day 1 infrastructure scope
- `GET /api/auth/me` — returns current user's id, email, and role from JWT claims; no DB call; Day 1 auth scope
- `docs/adr/005-clean-arch-vs-vsa.md` — documents Vertical Slice Architecture as evaluated alternative to Clean Architecture; explains why CA was chosen for this scope; Day 5 deliverable
- Rate limiting on `POST /api/auth/login` — fixed-window limiter (5 req/min per IP) using `AddRateLimiter` built into ASP.NET Core; **Day 2 scope**
- EF Core audit columns (`CreatedAt`, `UpdatedAt`) via `SaveChangesInterceptor` registered in DI; applied globally to all `IAuditableEntity` implementors: `Product`, `Cart`, `Order`; Day 1 infrastructure scope
- EF Core version corrected: EF Core 9 → **EF Core 10** throughout design doc
- Seed data: one admin user (`admin@example.com / Admin123!`) + 10 sample products seeded on startup; **Day 2 scope**
- Paginated search: `GET /api/products?page=&pageSize=&search=` returning `{ items: ProductDto[], total, page, pageSize }`; Day 2 added scope
- Redis cache: `IDistributedCache` for product queries; Day 3 added scope
- Order cancellation: `PATCH /api/orders/{id}/cancel`; extends `Order` aggregate with `Cancel()` behavior + status guard; Day 4 added scope
- 5 ADRs (Day 5 deliverables): `docs/adr/001-clean-architecture.md`, `docs/adr/002-cqrs-meditr.md`, `docs/adr/003-ddd-aggregates.md`, `docs/adr/004-no-microservices.md`, `docs/adr/005-clean-arch-vs-vsa.md`
