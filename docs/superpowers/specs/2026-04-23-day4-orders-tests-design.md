## Domain Layer

### New Value Objects / Entities

**`OrderStatus` enum** (`ECommerce.Domain/Enums/OrderStatus.cs`):
```
Pending | Completed | Cancelled
```

**`OrderItemData` record** (`ECommerce.Domain/Entities/OrderItemData.cs`):
Transient carrier used only in the `PlaceOrder` factory call — not persisted.
```csharp
record OrderItemData(ProductId ProductId, string ProductName, int Quantity, Money UnitPrice);
```

**`OrderItem` owned entity** (`ECommerce.Domain/Entities/OrderItem.cs`):
Snapshot stored inside the Order aggregate. Mirrors `CartItem` pattern.
- `Guid ProductId` — matches `Guid` type used in `CartItem` (no typed `ProductId` VO in codebase)
- `string ProductName` — snapshot at order time
- `int Quantity`
- `Money UnitPrice` — snapshot at order time

**`Order` aggregate root** (`ECommerce.Domain/Entities/Order.cs`):
- `Guid Id`
- `UserId UserId`
- `OrderStatus Status`
- `Money Total` — computed from items at creation
- `DateTime PlacedAt`
- `DateTime? CancelledAt`
- `IReadOnlyList<OrderItem> Items`

Methods:
- `static Order PlaceOrder(UserId userId, IEnumerable<OrderItemData> items)` — validates items non-empty, computes total (sum of `quantity × unitPrice`), raises `OrderPlaced` domain event, returns new Order with `Status = Pending`
- `void Cancel()` — throws `OrderAlreadyCompletedException` if `Status == Completed`; no-op (returns silently) if `Status == Cancelled` (idempotent); sets `Status = Cancelled`, `CancelledAt = DateTime.UtcNow` for `Status == Pending`

### Domain Events

**`OrderPlaced`** (`ECommerce.Domain/Events/OrderPlaced.cs`):
```csharp
record OrderPlaced(Guid OrderId) : IDomainEvent;
```
Raised inside `Order.PlaceOrder()`. Dispatched by `AppDbContext.SaveChangesAsync()` after DB write.

**`OrderPlacedNotificationHandler`** (`ECommerce.Application/Orders/Events/OrderPlacedNotificationHandler.cs`):
Stub handler. Logs the event via `ILogger`. No email, no inventory. Satisfies MediatR notification contract.

### Repository Interface

**`IOrderRepository`** (`ECommerce.Domain/Interfaces/IOrderRepository.cs`):
```csharp
Task<Order?> GetByIdAsync(Guid orderId, UserId userId);   // null if not found OR wrong user
Task<IReadOnlyList<Order>> GetByUserIdAsync(UserId userId);
void Add(Order order);
```

`GetByIdAsync` always filters by both `orderId` and `userId`. Returns `null` for both cases (not found, wrong user). Handler throws `OrderNotFoundException` → 404. Never 403 — do not leak order ID existence.

### New Domain Exceptions

**`OrderNotFoundException`** (`ECommerce.Domain/Exceptions/OrderNotFoundException.cs`):
```csharp
sealed class OrderNotFoundException() : NotFoundException("Order", string.Empty);
```
Extends `NotFoundException` (already maps to 404 in `ExceptionMiddleware`) — no middleware changes needed.

**`CartNotFoundException`** (`ECommerce.Domain/Exceptions/CartNotFoundException.cs`):
```csharp
sealed class CartNotFoundException(UserId userId) : DomainException($"Cart for user '{userId.Value}' was not found.");
```
Maps to 400 via `ExceptionMiddleware` (DomainException base → 400).

**`OrderAlreadyCompletedException`** (`ECommerce.Domain/Exceptions/OrderAlreadyCompletedException.cs`):
```csharp
sealed class OrderAlreadyCompletedException() : DomainException("Order is already completed and cannot be cancelled.");
```
Maps to 400.

> `OrderNotFoundException` extends `NotFoundException` — existing middleware maps it to 404 with no changes required.

## Application Layer

### Commands

**`PlaceOrderCommand`** (`ECommerce.Application/Orders/Commands/PlaceOrder/`):
```csharp
record PlaceOrderCommand(UserId UserId) : IRequest<Guid>;
```
Handler (`PlaceOrderCommandHandler`):
1. `ICartRepository.GetByUserIdAsync(userId)` — throw `CartNotFoundException` if null
2. `cart.Items` empty → throw `CartEmptyException`
3. For each cart item: `IProductRepository.GetByIdAsync(productId)` — throw `ProductUnavailableException` if `!product.IsActive`
4. Build `OrderItemData` list: snapshot `product.Name`, `cartItem.UnitPrice`, `cartItem.Quantity`
5. `Order.PlaceOrder(userId, orderItems)` — raises `OrderPlaced`
6. `_orderRepository.Add(order)`
7. `cart.Clear()`
8. `await _unitOfWork.CommitAsync()` — triggers domain event dispatch
9. Return `order.Id`

Validator (`PlaceOrderCommandValidator`): `UserId.Value != Guid.Empty`

**`CancelOrderCommand`** (`ECommerce.Application/Orders/Commands/CancelOrder/`):
```csharp
record CancelOrderCommand(Guid OrderId, UserId UserId) : IRequest;
```
Handler:
1. `IOrderRepository.GetByIdAsync(orderId, userId)` — throw `OrderNotFoundException` if null
2. `order.Cancel()` — throws `OrderAlreadyCompletedException` if completed
3. `await _unitOfWork.CommitAsync()`

Validator: `OrderId != Guid.Empty`, `UserId.Value != Guid.Empty`

### Queries

**`GetOrdersQuery`** (`ECommerce.Application/Orders/Queries/GetOrders/`):
```csharp
record GetOrdersQuery(UserId UserId) : IRequest<IReadOnlyList<OrderSummaryDto>>;
```
Handler: `IOrderRepository.GetByUserIdAsync(userId)` → map to `OrderSummaryDto` list.

**`GetOrderQuery`** (`ECommerce.Application/Orders/Queries/GetOrder/`):
```csharp
record GetOrderQuery(Guid OrderId, UserId UserId) : IRequest<OrderDetailDto>;
```
Handler: `IOrderRepository.GetByIdAsync(orderId, userId)` — throw `OrderNotFoundException` if null → map to `OrderDetailDto`.

### DTOs

## Infrastructure Layer

### EF Configuration

**`OrderConfiguration.cs`** (`ECommerce.Infrastructure/Persistence/Configurations/`):
- `HasKey(o => o.Id)`
- `UserId` value converter: `Guid ↔ UserId`
- `Status` stored as string: `.HasConversion<string>()`
- `Total` owned Money: `.OwnsOne(o => o.Total, b => { b.Property(m => m.Amount).HasColumnName("total_amount").HasPrecision(18,2); b.Property(m => m.Currency).HasColumnName("total_currency").HasMaxLength(3); })`
- Items owned collection: `builder.OwnsMany(o => o.Items, b => { ... })` — maps all fields, `UnitPrice` as nested owned Money
- Index: `ix_orders_user_id` on `UserId`

### Repository

**`OrderRepository.cs`** (`ECommerce.Infrastructure/Persistence/Repositories/`):
```csharp
GetByIdAsync(orderId, userId): 
    _context.Orders.Include(o => o.Items)
        .FirstOrDefaultAsync(o => o.Id == orderId && o.UserId == userId)

GetByUserIdAsync(userId):
    _context.Orders.Include(o => o.Items)
        .Where(o => o.UserId == userId)
        .OrderByDescending(o => o.PlacedAt)
        .ToListAsync()

Add(order): _context.Orders.Add(order)
```

### AppDbContext

Add `DbSet<Order> Orders` to `AppDbContext`.

### Migration

`AddOrder` — creates `orders` table + `order_items` owned table (or shadow table per EF owned collection strategy).

### DI Registration

Register `IOrderRepository → OrderRepository` in `InfrastructureServiceExtensions.AddInfrastructure()`.

## Constraints

- `OrderItem` stores product name and price as snapshot — never joins back to Product at read time
- `IOrderRepository.GetByIdAsync` always filters by both `orderId` and `userId` — null for both not-found and wrong-user
- `PlaceOrderCommandHandler` calls `IUnitOfWork.CommitAsync()` as final step — not the repository
- `AppDbContext.SaveChangesAsync()` dispatches `OrderPlaced` after DB write
- No `OrderId` in cart endpoints or PlaceOrder response body except `{ orderId }` — no caller-supplied IDs
- `CancelOrder` throws `OrderAlreadyCompletedException` on `Completed` status; already-`Cancelled` → no-op in `Order.Cancel()`, handler returns 204 silently
- All 13 design.md integration tests must pass in GitHub Actions CI (`ubuntu-latest`) before Day 5
